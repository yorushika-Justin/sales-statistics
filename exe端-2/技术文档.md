## 3. 核心架构

### 3.1 主要函数

| 函数 | 职责 |
|------|------|
| `read_xls(path, log_path)` | xlrd 读 `.xls`，按品牌汇总，返回 `({品牌: 金额}, 日期)` |
| `read_xlsx(path, log_path)` | openpyxl 读 `.xlsx`，逻辑同上 |
| `read_xls_items(path, log_path)` | 读 `.xls` 按品名提取明细，返回 `[(品名, 品牌, 条码, 数量, 金额, 库存), ...]` |
| `read_xlsx_items(path, log_path)` | 读 `.xlsx`，逻辑同上 |
| `draw_table(data, date_str, output_path)` | Pillow 画品牌汇总 PNG |
| `draw_table_detail(items, date_str, output_path)` | Pillow 画品名明细 PNG（7 列） |
| `_extract_date_from_xls(sh)` | 从 Row 2 提取日期，结束日 -1 天 |
| `_extract_date_from_xlsx(sh)` | 从 Row 3 提取日期（openpyxl 1-indexed） |
| `launch_gui()` | tkinterdnd2 拖拽窗口 |
| `show_error(msg, log_path, parent)` | 错误弹窗 + 写日志 |
| `main()` | 入口：有参数直接处理，无参数启动 GUI |

### 3.2 数据流

```
xls/xlsx 文件
    ↓
xlrd/openpyxl 读取
    ↓
提取日期（Row 2/3，结束日 -1 天）
    ↓
┌──────────────────┬──────────────────┐
│ 品牌汇总         │ 品名明细         │
│ 按品牌 sum       │ 逐行提取         │
│ 降序排序         │ 保持原顺序       │
│ draw_table()     │ draw_table_detail() │
└────────┬─────────┴────────┬─────────┘
         │                  │
         ▼                  ▼
    D:\销售汇总\<日期>\
    ├── <日期>.png        (品牌汇总，自动打开)
    └── <日期>_品名.png   (品名明细)
```

## 4. 关键约束

### 4.1 数据/格式
- **xlrd 1.2.0**：最后支持 `.xls` 的版本
- **openpyxl 3.1.5**：支持 `.xlsx`
- **表头检测**：前 20 行扫描，子串匹配"品牌名称"+"销售金额"
- **过滤规则**：空品牌 / "合计"开头 / "总"/"总计" / 金额为 0
- **日期规则**：结束日期 -1 天 = 实际统计日

### 4.2 路径/字体
- **字体硬编码**：`C:\Windows\Fonts\msyh.ttc`，缺失时崩溃
- **输出路径**：`D:\销售汇总\<日期>\`，自动创建，同名覆盖
- **D 盘必须存在**：否则 `os.makedirs` 失败

### 4.3 依赖

| 包 | 版本 | 用途 |
|----|------|------|
| Python | 3.13.12 | 运行环境 |
| xlrd | 1.2.0 | 读 `.xls` |
| openpyxl | 3.1.5 | 读 `.xlsx` |
| Pillow | 12.1.1 | 绘图 |
| tkinterdnd2 | 0.4.4.1 | GUI 拖拽 |
| pyinstaller | 6.19.0 | 打包 |

## 5. 错误日志分析指南

### 5.1 日志位置

| 场景 | 日志路径 |
|------|----------|
| exe 模式 | xls 文件同目录 `汇总错误.log` |
| 网页端模式 | `%TEMP%\sales_web_error.log` |

### 5.2 常见错误及原因

#### 错误 1：品名明细生成失败
```
品名明细生成失败（品牌汇总已正常生成）:
ValueError: 找不到 '品名' 列
```
**原因**：xls/xlsx 表头缺少"品名列"
**排查**：检查表头是否有"品名"列（不能是"商品名称"等变体）

#### 错误 2：品名明细生成失败
```
品名明细生成失败（品牌汇总已正常生成）:
ValueError: 找不到 '销售数量' 列
```
**原因**：表头缺少"销售数量"或"数量"列
**排查**：检查表头列名

#### 错误 3：字体缺失
```
FileNotFoundError: 找不到中文字体: C:\Windows\Fonts\msyh.ttc
```
**原因**：系统没有微软雅黑字体（极少见，Win10/11 自带）
**排查**：检查 `C:\Windows\Fonts\` 下是否有 `msyh.ttc`

#### 错误 4：输出目录无权限
```
PermissionError: [WinError 5] 拒绝访问: 'D:\销售汇总'
```
**原因**：D 盘存在但无写入权限
**排查**：以管理员身份运行，或检查 D 盘权限

#### 错误 5：D 盘不存在
```
FileNotFoundError: [WinError 3] 系统找不到指定的路径: 'D:\销售汇总'
```
**原因**：目标 PC 没有 D 盘
**排查**：检查磁盘分区

#### 错误 6：文件格式错误
```
xlrd.biffh.XLRDError: Unsupported format, or corrupt file
```
**原因**：文件后缀是 `.xls` 但实际是 `.xlsx` 格式（或反之）
**排查**：用 Excel 打开确认真实格式

## 9. 已知问题

### 9.1 品名明细可能失败
品名明细需要额外定位 4 列（品名/条码/数量/库存），如果表头列名不匹配会失败。品牌汇总不受影响。

### 9.2 文档注释可能过时
代码顶部注释的输出路径可能未同步更新，以 `OUTPUT_DIR` 常量为准。
