# 2026-07-02 系统信息检测脚本改进 — 网页端部署准备

## 任务背景

用户计划在新 PC（Win11 企业版，用户名 `song.jia.xin`）上部署网页端功能。已运行 `系统信息检测.py` 获取环境信息，发现需要补充检测项以便制定部署方案。

## 改动内容

### 1. 修复 `check_disk_space()`

| 问题 | 修复 |
|------|------|
| 路径拼接错误 | `C:` → `C:\`，`D:` → `D:\` |
| 原因 | `shutil.disk_usage()` 需要完整路径（含尾部斜杠） |

### 2. 改进 `check_python()`

| 新增检测项 | 用途 |
|------------|------|
| `pythonw.exe` 路径 | 开机自启 .bat 配置需要 |
| 查找逻辑 | 先查 Python 安装目录，再查 PATH 环境变量 |

### 3. 新增 `check_web_deps()`

检测网页端所需依赖包：

| 包名 | 用途 |
|------|------|
| Flask | Web 框架 |
| xlrd | 读 .xls |
| openpyxl | 读 .xlsx |
| Pillow | 绘图 |

输出：安装状态 + 版本号 + 安装命令提示

### 4. 新增 `check_web_files()`

检测网页端文件是否就位：

| 文件 | 用途 |
|------|------|
| `汇总脚本.py` | 网页端核心依赖（sys.path 引用） |
| `网页端-2/web_server.py` | Flask 主程序 |
| `网页端-2/config.json` | 配置文件 |
| `网页端-2/启动网页端.vbs` | 静默启动器 |
| `网页端-2/启动网页端.bat` | 调试启动器 |
| `网页端-2/开机自启.bat` | 开机自启管理 |

额外功能：读取并显示 `config.json` 配置内容

### 5. 改进 `generate_report()`

- 调用新增函数
- 报告新增三个章节：网页端依赖、网页端文件、config.json 配置
- 汇总检查新增：网页端依赖完整性、关键文件存在性
