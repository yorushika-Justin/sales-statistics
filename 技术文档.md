## 1. 项目概述

### 1.1 目标
将供应商每日从业务系统导出的销售明细 xls 文件，按"品牌名称"汇总"销售金额"，输出为 PNG 表格图片，供非技术用户查看与分享。

### 1.2 使用场景
- 用户每日从供应商门户导出当日销售明细（xls 格式）
- 拖拽 xls 到本工具 → 自动生成 `YYYY-MM-DD.png` 表格图片
- 生成的图片可直接发送、打印、归档

### 1.3 核心需求（来自 `提示文档.txt`）
1. 按"品牌名称"分组，sum "销售金额"
2. 输出表格含两列：品牌名称、销售金额（元）
3. 表格下方单独一行显示"总销售金额"
4. 异常数据保留原样（如"其它品牌"不归正）
5. 总金额应与原始数据合计一致

## 4. 脚本结构 (`汇总脚本.py`)

### 4.1 主要函数

| 函数 | 职责 |
|---|---|
| `extract_date_from_filename()` | 从 xls 文件名提取 `YYYY-MM-DD` 格式日期；失败用今天（现为 fallback） |
| `_extract_date_from_xls(sh)` | 从 xls sheet 的 Row 2 提取日期范围（优先使用）；**结束日期 -1 天**为实际最后完整统计日；`start == end`（当天导出当天，无滞后）→ 直接返回 `start`；`start != end` → 跨天显示 `start 至 (end-1)`。v1.5 修复了 `start==end` 边界 bug（详见 `工作记录 12.8`） |
| `read_xls(path, log_path)` | 读 xls，自动定位"品牌名称"列和"销售金额"列，按品牌 sum；返回 `(data, date_str)` 元组 |
| `read_xls_items(path, log_path)` | **v1.6 新增** 读 xls，按品名逐行提取明细（不汇总）；返回 `[(品名, 品牌, 条码, 数量, 金额, 库存), ...]`，保持原 xls 顺序 |
| `draw_table(data, date_str, output_path)` | Pillow 绘制品牌汇总表格 PNG（2 列），宽度自适应标题和品牌名 |
| `draw_table_detail(items, date_str, output_path)` | **v1.6 新增 / v1.6.1 修复** Pillow 绘制品名明细表格 PNG（7 列：行号/国际条码/品名/品牌/销售数量/销售金额/库存），宽度自适应；总计行统计总数量+总金额。**v1.6.1 修复**:3 处 `draw.rectangle` 右边界从 `x7` 改为 `x_right = x7 + stock_col_w`，补全库存列外框右边 |
| `show_error(msg, log_path, parent)` | tkinter 弹错误窗 + 写日志 |
| `launch_gui()` | 双击 exe 时启动拖拽窗口（需 tkinterdnd2） |
| `main()` | 入口：有参数 → 直接处理；无参数 → 启动 GUI 窗口 |

### 4.2 关键常量

| 常量 | 值 | 说明 |
|---|---|---|
| `FONT_PATH` | `C:\Windows\Fonts\msyh.ttc` | 微软雅黑 (Windows 自带) |
| `SIZE_TITLE / HEADER / BODY / TOTAL` | 32 / 26 / 22 / 24 | 各级字号 (像素) |
| `COL_BRAND_W / COL_AMOUNT_W` | 260 / 240 | 列宽 (像素) |
| `ROW_H` | 52 | 行高 (像素) |
| `PADDING` | 30 | 边距 (像素) |

### 4.3 数据定位策略
- 在前 20 行内扫描表头行
- 表头必须同时含 "品牌名称" 和 "销售金额"（均为子串匹配，兼容列名变体如"销售金额（元）"）
- 数据行跳过：空品牌 / 以"合计"开头的行 / 品牌为"总""总计"的行 / 金额为 0 的行
- 金额列含千分位逗号时自动清理再转换；转换失败的行计数并写日志
- 异常数据 (如品牌写"其它品牌"但品名前缀是"德佑") **保留原样，不归正**

## 7. 使用方法

### 7.1 普通用户 (推荐)

**方式A — 拖到图标 (最快，无需任何操作)**
1. 把 `dist\销售汇总.exe` 复制到任意位置 (可放桌面)
2. 把每日导出的 xls 文件**拖到 exe 图标上**
3. 自动弹出 PNG 表格

**方式B — 双击打开拖拽窗口**
1. 双击 `销售汇总.exe`
2. 在窗口中把 xls 文件拖入拖放区
3. 自动弹出 PNG 表格 + 状态栏显示生成结果

### 7.2 命令行
```powershell
& "D:\Code\project10-销售统计\dist\销售汇总.exe" "D:\Code\project10-销售统计\供应商日销售汇总_2026-06-05_195601.xls"
```

### 7.3 输出文件
- 路径：`D:\16678\相册\销售额\` (固定目录，不存在则报错提示)
- 命名：`YYYY-MM-DD.png` (日期优先从 xls 内部 Row 2 提取，结束日期 -1 天为实际统计日；失败则从文件名提取)

## 8. 已知限制

| 限制 | 影响 | 是否可解 |
|---|---|---|
| 只读第 1 个 sheet | 多门店多 sheet 时只能汇总第 1 个 | 可改, 遍历所有 sheet |
| 单门店 | 门店名来自第 4 行, 不参与汇总 | 同上 |
| 字体硬编码 | 字体丢失时启动失败 | 加 fallback (黑体 / 宋体) |
| 杀毒误报 | pyinstaller 通病 | 加数字签名 (需证书) |
| 中文路径文件名 | pyinstaller 输出有乱码但不影响功能 | 无需处理 |
| 行数自适应 | 超过 ~20 个品牌时图片很高 | 已自适应, 暂无问题 |
| **压缩限制 (34 MB)** | 比预期 15 MB 大 | 见 8.1 |

### 8.1 压缩限制 (已评估, 接受 34 MB)

经测试, `--exclude-module pygame` **无法生效** —— pygame 是被 PIL/Pillow 在 Windows 上的钩子链**隐式 import** 的 (打包日志可见 `D:\Python\Lib\site-packages\pygame\pkgdata.py:25` 触发的 `pkg_resources` 警告和 "Hello from the pygame community" 输出)。

**最终决策**: 接受 34 MB。
- 34 MB 对单文件 Python 应用是正常水平
- 强排风险高 (方案 A 可能误删 PIL 依赖)
- UPX 压缩需额外工具且有兼容问题
- **功能稳定 > 体积**

**如未来要重试压缩**:
- **方案 A**: 改 `汇总脚本.spec` 的 `excludes=['pygame', 'pygame.*']` 强排, 预期 ~15 MB, 需重测图生成
- **方案 B**: 装 UPX (`pip install upx`), 打包时加 `--upx-dir=<upx 路径>`, 预期 ~10 MB

## 10. 维护指南

### 10.1 修改代码后重打包
```powershell
cd D:\Code\project10-销售统计
# 改 汇总脚本.py 后:
pyinstaller --onefile --noconsole --exclude-module pygame --clean --noconfirm --name=销售汇总 汇总脚本.py
```

### 10.2 xls 格式变化时改哪里
- **列名改了** (如"销售金额"变成"销售总额") → 改 `read_xls()` 中的列名匹配
- **表头位置变了** (如从第 6 行变到第 8 行) → 已支持前 20 行扫描, 无需改
- **新增 sheet** → 改 `read_xls()` 用 `wb.sheet_by_index(i)` 遍历

### 10.3 字体问题
- 若 `msyh.ttc` 缺失, 把 `FONT_PATH` 改为 `C:\Windows\Fonts\simhei.ttf` (黑体)
- 终极方案: 把字体文件打包进 exe, 用 `sys._MEIPASS` 路径访问

### 10.4 调试模式
- 临时去掉 `--noconsole` 可看到 print 输出, 便于排错
- 排错完毕再恢复 `--noconsole` 重新打包
