# 程序端（Render 云部署）技术文档

## 1. 概述

将销售汇总工具部署到 Render.com 免费云平台，实现**手机端上传 xls/xlsx → 生成 PNG → 下载到手机**。

- **部署地址**: `https://sales-statistics-rdle.onrender.com`
- **代码仓库**: `https://github.com/yorushika-Justin/sales-statistics`
- **版本**: v1.0（2026-07-04）
- **费用**: 全部免费（Render Free Tier）

## 2. 架构

```
手机夸克浏览器
    │
    ▼
Render.com（云服务器，免费层）
    │
    ├─ 程序端/web_server.py  ← Flask 入口
    ├─ 程序端/汇总脚本.py    ← 适配层（mock tkinter + 覆盖配置）
    ├─ 程序端/msyh.ttc       ← 微软雅黑字体（19MB）
    └─ 汇总脚本.py            ← 原版核心逻辑（不修改）
```

## 3. 文件清单

| 文件 | 说明 |
|---|---|
| `程序端/web_server.py` | Flask 服务，路由 /、/upload、/download、/health |
| `程序端/汇总脚本.py` | 适配层，从原版导入核心函数，覆盖 OUTPUT_DIR/FONT_PATH，mock tkinter |
| `程序端/msyh.ttc` | 微软雅黑字体，嵌入仓库（Render 无法 apt-get） |
| `requirements.txt` | Python 依赖：flask、xlrd、openpyxl、Pillow |
| `Procfile` | Render 启动命令：`web: python 程序端/web_server.py` |
| `render.yaml` | Render 配置（部分字段手动设置更准确） |

## 4. 关键设计

### 4.1 适配层模式（不修改原版）

`程序端/汇总脚本.py` 通过以下方式复用原版逻辑：

1. `sys.path.insert` 将项目根目录加入 import 路径
2. `from 汇总脚本 import read_xls, draw_table, ...` 导入核心函数
3. 覆盖模块级变量（OUTPUT_DIR、FONT_PATH）
4. 提供 `process_xls()` 封装入口

### 4.2 tkinter Mock

原版 `汇总脚本.py` 顶部 `import tkinter`，Linux 服务器无 Tk 库会报错。适配层在导入前 mock：

```python
if os.name != 'nt':
    sys.modules['tkinter'] = _tk_mock
    sys.modules['tkinter.messagebox'] = _tk_mock.messagebox
```

### 4.3 字体处理

Render 免费层无 sudo 权限，无法 `apt-get install fonts-wqy-zenhei`。解决方案：将 `msyh.ttc`（19MB）直接放入仓库，`FONT_PATH` 指向仓库内文件。

### 4.4 下载机制

- `/download/<path>` 路由：`?dl=1` 触发下载，否则内联显示（给 `<img>` 用）
- 前端：图片预览 + "保存" 按钮（`?dl=1`）
- 安全：路径穿越检查（`normpath` + `startswith`）

### 4.5 环境变量

| 变量 | 默认值 | 说明 |
|---|---|---|
| `PORT` | 5000 | Render 自动分配，无需手动设 |
| `OUTPUT_DIR` | `~/output/sales` | PNG 输出目录 |
| `FONT_PATH` | 仓库内 `msyh.ttc` | 字体路径 |

## 5. 用户体验

### 手机端流程

1. 夸克打开 `https://sales-statistics-rdle.onrender.com`
2. 点击选择区域 → 选择 xls/xlsx 文件
3. 等待处理（首次冷启动约 30 秒）
4. 页面显示品牌汇总 + 品名明细图片预览
5. 点击「保存品牌汇总」「保存品名明细」→ 下载到手机

### 限制

- 冷启动：Render 免费层 15 分钟不活跃后休眠，首次访问约 30 秒
- 文件大小：10MB 限制（`MAX_CONTENT_LENGTH`）
- 速度：免费层 CPU 有限，大文件处理较慢

## 6. Render 免费层说明

- **750 小时/月** — 足够单服务 24/7 运行
- **无硬性到期日** — 政策可能调整，需关注官方
- **15 分钟休眠** — 不活跃自动休眠，请求触发唤醒
- **无需电脑** — 代码在云服务器，手机即可使用

## 7. 部署步骤（备忘）

1. GitHub 推送代码
2. Render → New Web Service → 连接 GitHub 仓库
3. Build Command: `pip install -r requirements.txt`
4. Start Command: `python 程序端/web_server.py`
5. 等待部署完成，访问公网地址
